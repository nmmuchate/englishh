## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Quasar class | Usage |
|-------|-------|-------------|-------|
| xs | 4px | q-pa-xs / q-ma-xs | Icon gaps, lock icon overlay inset |
| sm | 8px | q-pa-sm / q-ma-sm | Chip padding, card grid gutter (`q-col-gutter-sm`) |
| md | 16px | q-pa-md / q-ma-md | Page default padding, card inner padding |
| lg | 24px | q-pa-lg / q-ma-lg | Page outer horizontal padding (`q-px-lg`), section spacing |
| xl | 32px | q-mb-xl | Gap between card grid and CTA row |
| 2xl | 48px | — | Major section breaks |
| 3xl | 64px | — | Page-level spinner centering area |

Exceptions:
- Session type cards in 2×2 grid: minimum tap target height 88px per card (two-row label + icon comfortably above 44px minimum)
- Loading spinner area: `q-inner-loading` uses `q-spinner-dots size="64px"` — 64px matches established Phase 15 pattern
- Brief page "Start Session" CTA: `size="lg"` QBtn with full-width — adds implicit 12px vertical padding per Quasar's `lg` size

> Source: STATE.md `[03-01]` — stat card grid uses `q-col-gutter-sm`; PlacementResultPage uses `q-pa-lg` page wrapper. 2×2 card layout recommended by CONTEXT.md D-Claude's Discretion.

## Typography

| Role | Size | Weight | Line Height | Quasar class |
|------|------|--------|-------------|-------------|
| Body | 14px | 400 (regular) | 1.5 | text-body2 |
| Caption / hint | 14px | 400 (regular) | 1.4 | text-caption — distinguished by `color: var(--q-grey-6)` |

> Source: Phase 15 UI-SPEC typography table — 4 roles, 2 weights (400 + 700), unchanged for Phase 16.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Dark: `#151d15` / Light: `#ffffff` | Page background; FullscreenLayout pages inherit this |
| Secondary (30%) | Dark: approx `#1e2a1e` / Light: `#f6f7f6` | QCard flat card backgrounds (session type cards, brief sections) |
| Accent (10%) | `#4cae4f` (--primary / var(--q-primary)) | Unlocked card border, primary CTA buttons, lock icon color override |
| Accent-orange | `#FF6B35` (--accent-orange) | Not used in Phase 16 (no strengths/weaknesses display) |
| Destructive | `#ef5350` (Quasar `text-negative` / `bg-red-1`) | Error state banners (generateSessionPlan failure) |

Accent reserved for:
- `q-btn color="primary"` — "Start Session" CTA on ScenarioBriefPage
- Unlocked `SessionTypeCard` border: `border: 2px solid var(--q-primary)` on the active/unlocked card variant
- QChip on SessionTypeCard showing session duration: `color="primary" text-color="white"` for unlocked sessions
- Loading spinner: `q-spinner-dots color="primary"`

Locked card treatment (from CONTEXT.md D-02):
- `opacity: 0.5` on the entire locked card
- Lock icon overlay: `q-icon name="lock" size="20px"` at top-right corner with `position: absolute; top: 8px; right: 8px` — color defaults to grey-6, no primary accent
- No border accent on locked cards (opacity handles the visual distinction)

> Source: app.css design tokens; STATE.md `[Init]` decision; CONTEXT.md D-02 lock state specification; Phase 15 UI-SPEC color table (extended, not changed).

## Screen-by-Screen Contract

### SessionTypeSelectPage.vue

**Route:** `/sessions` (FullscreenLayout — no bottom nav)

**Layout:** Single scrollable column with `q-pa-md` wrapper, `max-width: 430px` via `.app-container`.

**Sections (top to bottom):**

1. **Page heading**
   - `text-h6 text-weight-bold q-mb-xs` — "Choose Your Session"
   - `text-body2 text-grey-6 q-mb-lg` — "Select the type of English practice you want today."

2. **Session type grid**
   - `div class="row q-col-gutter-sm q-mb-xl"`
   - 4 cards in 2×2 layout: `div class="col-6"` per card
   - Each card uses `<SessionTypeCard>` with props: `type`, `locked`, `icon`, `name`, `description`, `duration`

3. **Loading overlay**
   - `q-inner-loading :showing="isGenerating"`
   - `q-spinner-dots size="64px" color="primary"`
   - Below spinner: `p class="text-body2 text-grey-6 q-mt-md text-center"` — "Preparing your session..."

4. **Error state**
   - `q-banner class="bg-red-1 text-red-9 q-mb-lg" rounded` shown when `generateError` is set
   - "Retry" primary button + "Back" flat button

**States:**

| State | UI |
|-------|----|
| Default | Grid of 4 cards; A1-A2 user sees cards 3+4 at opacity 0.5 |
| Tapping unlocked card | `isGenerating = true` → loading overlay |
| Tapping locked card | `$q.notify` snackbar; no navigation |
| generateSessionPlan success | Navigate to `/session/brief` |
| generateSessionPlan error | `isGenerating = false`; error banner shown; retry stays on same page |

### SessionTypeCard.vue

**Layout:** `q-card flat class="session-type-card"` with `position: relative` for lock icon overlay.

**Unlocked card:**
```
q-card flat
  position: relative
  border-radius: var(--radius-md) [16px]
  border: 2px solid var(--q-primary)
  padding: 16px (q-pa-md)
  cursor: pointer
  min-height: 88px

  [top area]
  q-icon :name="icon" size="28px" color="primary" — top-left icon

  [body]
  p.text-subtitle1.text-weight-bold — type name
  p.text-body2.text-grey-6 — type description (1 line max, truncate)

  [bottom]
  q-chip dense color="primary" text-color="white" — duration (e.g. "15–20 min")
```

**Locked card:**
- All above structure but wrapped in `div style="opacity: 0.5; pointer-events: none; position: relative"`
- Parent `div` retains pointer-events + adds click handler for the lock notification
- Lock overlay: `q-icon name="lock" size="20px" color="grey-6" style="position:absolute; top:8px; right:8px"`
- No primary border — use `border: 2px solid transparent` or `border: 1px solid var(--q-separator-color)` instead
- Lock subtitle: `p class="text-caption text-grey-5"` — "Available at B1"

**Session type definitions:**

| Type | Icon | Name | Description | Duration | Min CEFR |
|------|------|------|-------------|----------|----------|
| free-talk | `chat` | Free Talk | Open conversation on any topic | 15–20 min | A1 |
| scenario | `work` | Scenario | Role-play a real-world situation | 15–20 min | A1 |
| story-builder | `menu_book` | Story Builder | Build a story together with AI | 20–25 min | B1 |
| debate | `gavel` | Debate | Argue your position on a topic | 20–25 min | B1 |

> Icon choices from CONTEXT.md "Claude's Discretion" — chat/work/menu_book/gavel from Material Design Icons set (all available via Quasar q-icon).

## Interaction States

### SessionTypeSelectPage — Type Selection Flow

```
Load page → read CEFR level from placementStore.finalResult.overallLevel
          → A1/A2: story-builder + debate render locked
          → B1+: all 4 unlocked

[tap unlocked card]
  → isGenerating = true → loading overlay shown
  → call generateSessionPlanFn({ type, userProfile, skillGaps, sessionHistory })
  → [success] → store plan in sessionStore → navigate to /session/brief
  → [error] → isGenerating = false → error banner shown

[tap locked card]
  → $q.notify({ message: '{Type Name} unlocks at B1', type: 'warning', position: 'top' })
  → no navigation
```

### ScenarioBriefPage — Start Session Flow

```
Land on page → read plan from sessionStore (already populated)
             → render topic, role, context, objectives

[tap Start Session]
  → router.push({ name: 'session' })
  → SessionPage reads sessionId from sessionStore (already set by generateSessionPlan)
  → sendMessage + endSession unchanged

[tap Choose Different Type]
  → router.push({ name: 'sessions' })
```

### DashboardPage — FAB Flow (Modified)

```
[tap Start Session FAB]
  → router.push({ name: 'sessions' })  [was: { name: 'session' }]
```

## Accessibility

- All `SessionTypeCard` components must have `role="button"` and `tabindex="0"` — they are `q-card` not `q-btn` but must be keyboard-accessible
- Locked cards: add `aria-disabled="true"` and `aria-label="{Type Name} — locked, available at B1"` on the click wrapper div
- `$q.notify` snackbar for locked taps: Quasar's QNotify renders with `role="alert"` by default — no additional ARIA needed
- `q-inner-loading` uses `aria-busy="true"` on the parent — Quasar handles this natively via `q-inner-loading`'s `showing` prop
- Objective check icons in ScenarioBriefPage: `aria-hidden="true"` on `q-icon` since adjacent text conveys the content
- "Start Session" button: no additional ARIA needed — label is self-describing

---

## Animation & Motion

- Card tap feedback: Quasar's built-in `ripple` on `q-card` — enabled by default; no custom animation
- Loading overlay: no fade-in; `q-inner-loading` renders immediately when `isGenerating = true`
- Locked card opacity: CSS `opacity: 0.5` only — no transition on lock state (static at load time)
- ScenarioBriefPage: no entry animation; content renders immediately from session store data
- No custom keyframe animations introduced in this phase (Phase 15 mic pulse animation is not present here)
