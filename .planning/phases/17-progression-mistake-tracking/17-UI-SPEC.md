## Component Inventory

### Level-Up Banner (new — inline in FeedbackPage.vue)

**Trigger:** `sessionStore.levelUps` is non-empty on FeedbackPage mount.

**Structure:**
```
div.level-up-banner (v-if="levelUps.length > 0")
  q-icon[sym_o_trending_up] — 20px, color primary
  span — "Your {skill} just leveled up to {level}!"
  (one banner per level-up; stack vertically with q-gutter-sm if multiple)
```

**Visual spec:**
- Background: `rgba(76, 174, 79, 0.12)`
- Border: `1px solid rgba(76, 174, 79, 0.25)`
- Left border accent stripe: `3px solid #4cae4f`
- Border-radius: `--radius-sm` (0.5rem / 8px)
- Padding: `q-px-md q-py-sm` (16px / 8px)
- Icon: `sym_o_trending_up`, 20px, color `primary`
- Text: `text-subtitle2` weight 600, color white (dark) / `--text-deep-slate` (light)
- Skill name: capitalised — "Vocabulary", "Grammar", "Speaking", etc.
- Level name: rendered in `color: primary` (`#4cae4f`) within the sentence

**Insertion point:** Immediately before the hero card `.hero-card` div, inside `.feedback-scroll`. Separated from hero card with `q-mb-md` (16px).

**Animation:** `transition: opacity 0.3s ease-in` on mount; no dismiss button (banner is informational, disappears on next session start via `levelUps` reset in `startSession()`).

**Multiple level-ups:** Render one banner row per `levelUps` entry using `v-for`. Stack with `q-gutter-sm` (8px gap).

**Light mode override:**
```css
.body--light .level-up-banner {
  background: rgba(76, 174, 79, 0.08);
  border-color: rgba(76, 174, 79, 0.2);
  color: var(--text-deep-slate, #131613);
}
```

### No New Pages or Routes

Phase 17 creates no new pages and no new routes. All backend work (endSession extension, getWeeklyReview Cloud Function, mistake persistence) has zero visual output beyond the level-up banner.

## Interaction Contract

| Interaction | Behaviour |
|-------------|-----------|
| FeedbackPage mounts with `levelUps` non-empty | Banner(s) fade in above hero card via CSS opacity transition (300ms ease-in) |
| FeedbackPage mounts with `levelUps` empty | No banner rendered; page layout unchanged |
| Multiple skills level up in same session | One banner per skill, stacked vertically, ordered by skill array index |
| User navigates away and starts new session | `startSession()` resets `levelUps.value = []`; banners will not reappear |
| getWeeklyReview Cloud Function called | No UI output in Phase 17; function is deployed but no UI trigger exists until Phase 19 |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Level-up banner (single skill) | "Your {Skill} just leveled up to {Level}!" — e.g. "Your Grammar just leveled up to B1!" |
| Level-up banner (max level reached) | "Your {Skill} has reached the top level — C2!" |
| Empty state | Not applicable — banner only appears when `levelUps` is non-empty |
| Error state (skill update fails) | Silent best-effort — no user-facing error; session scores still display normally |
| Destructive confirmation | None — no destructive actions in this phase |

**Skill name capitalisation:** vocabulary → Vocabulary, reading → Reading, listening → Listening, grammar → Grammar, speaking → Speaking, writing → Writing.

**Level abbreviations:** A1, A2, B1, B2, C1, C2 — always uppercase, no expansion.

**Source:** CONTEXT.md D-03 (level-up banner design decision); CEFR level sequence from Claude's Discretion.
