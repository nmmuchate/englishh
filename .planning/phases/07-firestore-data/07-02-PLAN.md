**B) src/pages/OnboardingPage.vue — add Firestore write in handleComplete():**

Current handleComplete():
```javascript
function handleComplete() {
  authStore.completeOnboarding()
  router.push({ name: 'dashboard' })
}
```

Replace with:
```javascript
async function handleComplete() {
  const { doc, updateDoc, serverTimestamp } = await import('firebase/firestore')
  const { db } = await import('boot/firebase')
  const userRef = doc(db, 'users', authStore.uid)
  await updateDoc(userRef, {
    onboardingCompleted: true,
    currentLevel:        'B1',
    levelProgress:       0,
    freeSessionUsed:     false,
    updatedAt:           serverTimestamp()
  })
  authStore.completeOnboarding()
  router.push({ name: 'dashboard' })
}
```

IMPORTANT: Use dynamic `import()` (as shown) to avoid adding top-level imports that would introduce a firebase import at module load time — this keeps the onboarding page from loading Firebase before boot/firebase.js initializes. Alternatively, if top-level import is cleaner:

Top-level import approach (preferred — boot/firebase.js is already initialized by the time OnboardingPage mounts):
Add at top of `<script setup>`:
```javascript
import { doc, updateDoc, serverTimestamp } from 'firebase/firestore'
import { db } from 'boot/firebase'
```

And change `function handleComplete()` to `async function handleComplete()` with `await updateDoc(...)`.

Use the top-level import approach (cleaner, avoids dynamic import complexity). The `handleComplete` call site is the `@click="handleComplete"` button — Vue handles async event handlers correctly.

Also change the button's `@click` to use the async handler: `@click="handleComplete"` — no change needed since Vue handles this transparently.

CRITICAL: Use `updateDoc` not `setDoc` — the users/{userId} document already exists (created by createUserProfile during signInWithGoogle). setDoc without {merge:true} would wipe all fields including email, subscriptionStatus, etc.
