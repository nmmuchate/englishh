## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Quasar class | Usage |
|-------|-------|-------------|-------|
| xs | 4px | q-pa-xs / q-ma-xs | Icon gaps, inline padding |
| sm | 8px | q-pa-sm / q-ma-sm | Compact element spacing, q-mb-sm |
| md | 16px | q-pa-md / q-ma-md | Stage wrapper default padding (q-pa-md), card inner padding |
| lg | 24px | q-pa-lg / q-ma-lg | Section padding, progress bar top area (q-px-lg q-pt-lg) |
| xl | 32px | q-pa-xl | Large icon centering area |
| 2xl | 48px | — | Major section breaks |
| 3xl | 64px | — | Page-level spacing |

Exceptions: Touch targets for mic button must be minimum 44px height; QBtn with padding `12px 16px` used on answer options (established pattern from Vocabulary/Grammar/Listening stages).

> Source: Existing stage components (VocabularyStage, GrammarStage, ListeningStage) establish q-pa-md as the stage wrapper standard, q-mb-lg between major sections, and explicit style `padding: '12px 16px'` on answer buttons.

## Screen-by-Screen Contract

### SpeakingStage.vue

**Layout:** Single scrollable column inside `q-pa-md`.

**States:**

| State | UI |
|-------|----|
| Loading | 3 q-skeleton rows (8px bar + 80px card + 48px button) |
| Error | q-banner bg-red-1 + "Retry" primary button + "Skip this stage" flat button |
| Ready (before first exchange) | Stage label "Speaking" + progress counter + AI prompt card + mic button (or text input) |
| Recording | Mic button shows active pulsing ring (`animation: pulse 1.2s infinite`; border: 2px solid var(--primary)) + interim transcript text in grey |
| Submitted (waiting AI response) | q-inner-loading spinner over the exchange area |
| AI responded | AI bubble (q-card, bg-grey-2 dark:bg-grey-9, left-aligned) rendered; mic/input re-enabled for next turn |
| Final exchange complete | "Submit Speaking" primary pill button appears |
| All exchanges done | Emit complete |

**Exchange bubbles:**
- User bubble: right-aligned, `bg-primary text-white`, border-radius var(--radius-lg) 24px on left side, 4px on right
- AI bubble: left-aligned, bg-grey-2 (light) / bg-grey-9 (dark), border-radius var(--radius-lg) 24px on right side, 4px on left
- Padding inside bubble: 12px 16px (q-pa-sm q-pa-md)

**Mic button:**
- Circular QBtn, size 56px, color="primary" when idle, colour="negative" when recording
- Icon: `mic` (idle) / `stop` (recording)
- Position: centered, `q-mt-lg`
- Touch target: 56px — above 44px minimum

**Text fallback:**
- QInput outlined rounded, label="Type your response", full-width
- Send QBtn color="primary" unelevated rounded no-caps, label="Send"
- Shown when `window.SpeechRecognition` and `window.webkitSpeechRecognition` are both unavailable
- Same pattern as SessionPage.vue (Phase 8 decision [08-02])

**Progress:** `text-caption text-grey-6` label "Speaking" + `text-caption text-primary text-weight-bold` exchange counter "Exchange 1 of 3". `q-linear-progress` height 6px color="primary".

### WritingStage.vue

**Layout:** Single scrollable column inside `q-pa-md`.

**States:**

| State | UI |
|-------|----|
| Ready | Stage label + progress (shows "Writing — 1 of 1") + prompt card + textarea + character counter + submit button |
| Submitting | q-inner-loading over textarea + submit button disabled |
| Error | q-banner bg-red-1 + retry + skip |

**Prompt card:** `q-card flat bordered q-pa-md q-mb-lg` with prompt text at `text-subtitle1 text-weight-medium` line-height 1.5.

**Textarea:** `q-input type="textarea" outlined rounded` with:
- `label="Write your response here..."`
- `autogrow` so it expands as user types
- `counter` showing character count
- `minlength` hint: `text-caption text-grey-5 q-mt-xs` "Aim for 2-3 sentences"
- Full-width

**Submit button:** `q-btn label="Submit Response" color="primary" unelevated rounded no-caps class="full-width q-mt-md"` — disabled until textarea has at least 20 characters.

**Skip link:** Same flat grey pattern as other stages.

## Interaction States

### Speaking Stage — Voice Flow

```
Idle → [tap mic] → Recording (pulsing ring, "stop" icon)
     → [tap stop / silence detection] → Interim transcript frozen
     → [AI processes] → Loading overlay
     → [AI responds] → AI bubble rendered, next exchange enabled
     → [3-4 exchanges complete] → "Submit Speaking" button appears
     → [tap Submit] → emit('complete', result)
```

### Writing Stage — Submit Flow

```
Blank → [type] → character count updates, submit button enabled at 20+ chars
      → [tap Submit Response] → submitting state, overlay shown
      → [complete] → emit('complete', result)
```

### PlacementResultPage — Calculate Flow

```
Mounted → call calculatePlacement Cloud Function
        → isCalculating = true → spinner shown
        → [function returns] → render results
        → [tap CTA] → navigate to session type select (Phase 16) or /app/dashboard
```

### Skip Flow (all stages)

```
[tap "Skip this stage"] → emit('skip')
→ OnboardingPage sets stage result to { score: 0, level: 'B1', skipped: true }
→ advance to next stage
```

## Accessibility

- Mic button must have `aria-label` bound to idle/recording state (dynamically)
- All interactive QBtn elements use `no-caps` — avoid ALL-CAPS which reduces readability
- QLinearProgress must have `aria-label` e.g. "Stage progress" and `aria-valuenow` equivalent via Quasar `:value` prop (Quasar handles ARIA for q-linear-progress natively)
- PlacementResultPage CEFR badge: include `role="img"` with `aria-label="Overall level: B1 Intermediate"` on the badge card
- QInput textarea in WritingStage: `aria-label="Write your response"` via `input-attrs` prop

---

## Animation & Motion

- Progress bar: no custom animation; Quasar's built-in transition on q-linear-progress value change is sufficient
- Mic button active state: CSS keyframe pulse ring — `animation: pulse 1.2s ease-in-out infinite` using `box-shadow: 0 0 0 8px rgba(76,174,79,0.2)` — matches the `@4cae4f` primary green
- Exchange bubble entry: `v-for` with `:key="idx"` renders immediately; no animation (keeps consistency with ListeningStage which has none)
- PlacementResultPage skill bars: use `v-bind:value` computed from finalResult; Quasar animates width change on component mount naturally via its internal transition
